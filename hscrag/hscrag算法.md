这是一个**层次化结构感知的智能文本分块算法**，专为RAG系统设计。我来详细讲解其核心算法逻辑：

## 1 一、算法目标

在**保持文档结构完整性**和**控制分块大小**之间找到平衡，生成高质量的RAG检索单元。

## 2 二、核心数据结构

```
文档 → [治理块(Block)] → [RAG块(Chunk)]
```

- **治理块**：已结构化的最小单元（段落、标题、表格等）
- **RAG块**：最终输出的检索单元，包含上下文信息

## 3 三、算法流程

### 3.1 主流程（`chunk_document`）

```
1. 提取治理块列表
2. 自适应配置调整（根据文档特征）
3. 构建初始块（核心逻辑）
4. 合并短块
5. 重新编号输出
```

### 3.2 核心构建逻辑（`_build_initial_chunks`）

这是一个**增量缓冲算法**：

```
缓冲块列表 buffer = []
缓冲文本 buffer_texts = []
缓冲token数 buffer_tokens = 0

对每个治理块 unit:
    计算文本和token数
    
    情况1: 受保护块（表格/代码/公式）且超大
        → 立即作为独立块输出
    
    情况2: 普通块且超大
        → 按句子分割成多个块
    
    情况3: 正常块
        → 调用边界评分判断是否分割
        → 如果决定分割，flush缓冲区并开始新块
        → 否则加入缓冲区继续累积
```

### 3.3 边界评分算法（`_score_boundary`）

这是算法的**核心创新**，综合三个信号决定是否分割：

#### 3.3.1 计算公式：
```
score = 0.45×结构信号 + 0.35×语义距离 + 0.20×长度压力
```

#### 3.3.2 三个信号详解：

**① 结构信号（0-1）**
```
层级变化权重：
- 顶级标题变化 → 1.0（强分割信号）
- 子标题变化   → 0.85
- 不同章节     → 0.70
- 受保护块开头 → 0.45
- 块类型变化   → 0.25
- 无变化       → 0.0
```

**② 语义距离（0-1）**
- 使用**词袋模型的余弦相似度**   忽略词的顺序和语法，只统计每个词在文档中出现的频率
- 计算当前块与最近N个块的语义差异
- 距离越大，越应该分割

**③ 长度压力（0-1）**
```
buffer_tokens / target_tokens
```
- 缓冲区越长，分割压力越大

#### 3.3.3 分割决策条件（满足任一即分割）：

```
1. 超出最大长度 → 强制分割
2. 语义距离 > 0.72 且 缓冲≥180tokens 且 长度压力≥0.65 → 语义边界
3. 章节变化 且 缓冲≥180tokens 且 评分≥0.52 → 章节边界
4. 评分≥0.62 且 缓冲≥180tokens → 综合评分边界
5. 缓冲≥目标token数 且 评分≥0.52 → 目标长度边界
```

### 3.4 自适应配置（`_adaptive_config_for_document`）

根据文档特征动态调整阈值，三种模式：

#### 3.4.1 模式识别特征：
```python
文档统计特征：
- 平均块大小
- 标题变化率
- 语义连续性
- 受保护块比例
- 短块比例
```

#### 3.4.2 三种模式：

| 模式 | 触发条件 | 调整策略 |
|------|----------|----------|
| **结构保留模式** | 标题变化多、语义跳跃大 | 降低阈值，更易分割（保持结构） |
| **高上下文覆盖** | 块短小、语义连续 | 提高阈值，增加目标token数（更多上下文） |
| **上下文覆盖** | 中等上下文需求 | 适度提高阈值和目标大小 |

### 3.5 语义相似度计算（`_semantic_similarity`）

使用**局部词袋余弦相似度**：

```python
1. 取缓冲区最后N个块（默认3个）
2. 提取所有token（支持中英文）
3. 构建词频向量
4. 计算余弦相似度
5. 返回 1 - 相似度 作为语义距离
```

### 3.6 合并策略（`_merge_short_chunks`）

防止产生过短的块：

```python
条件：
- 当前块 < 最小token数（默认180）
- 与下一个块合并不超过最大token数（默认900）
- 属于同一顶级章节
→ 合并两块
```

## 4 四、算法优势

### 4.1 **多信号融合**
不是简单按固定大小切分，而是综合结构、语义、长度三个维度

### 4.2 **结构感知**
尊重文档的层次结构，不破坏标题-内容关系

### 4.3 **自适应**
根据文档特征自动调整参数，无需手动调优

### 4.4 **可追溯**
每个分割决策都记录详细信号，便于调试和评估

### 4.5 **保护特殊块**
表格、代码、公式等完整性块尽量保持完整

## 5 五、实际应用示例

假设一篇技术文档：
```
# 第一章 (标题变化)
  1.1 背景 (子标题)
    段落1 (语义连贯)
    段落2 
  1.2 方法 (子标题变化，触发分割)
    段落3
    代码块 (受保护，单独处理)
    段落4
```

算法会：
1. 在1.1和1.2之间分割（结构信号强）
2. 代码块单独成块（受保护）
3. 段落1-2合并在一个块（语义相似）
4. 控制每个块在180-900 tokens之间

## 6 六、性能考虑

- **时间复杂度**：O(n×w)，n为块数，w为语义窗口大小
- **空间复杂度**：O(n)，主要存储缓冲区
- **优化点**：增量计算，避免重复处理

这个算法的核心价值在于**生成语义完整、结构清晰、大小适中的RAG检索单元**，从而提升下游检索和生成的质量。
